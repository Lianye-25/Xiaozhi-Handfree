# HANDSFREE 五维方向语义规范

## 概述

HANDSFREE 将信息处理分解为两条协作轴：纵向的"人"轴（↑↓ 上下级管理）和横向的"机"轴（←→ AI Agent 管理）。用户位于十字中枢，根据任务的性质和流向，将工作分派到不同方向。

---

## ↑ UP — 上报（Report）

### 语义定义
用户将工作成果、报告、消息提交给组织架构中比自己级别高的人。

### 适用场景
- 完成阶段性工作后向上级汇报进展
- 提交需要审批的文档或方案
- 将重要信息同步给管理层
- 请求上级决策或指导

### 关键词
`上报` `汇报` `提交` `报告` `告诉` `通知` `呈报` `报给` `发给` `反馈给`

### 匹配联系人
`contacts.json` 中的 `superiors` 列表

### 反例（不应归为 UP 的情况）
- "告诉小王去开会" → 这是 DOWN（小王是下属）
- "提交代码到仓库" → 非人际汇报，不匹配任何方向

---

## ↓ DOWN — 委派（Delegate）

### 语义定义
用户将任务下达给下属或执行层人员，完成人际层面的任务分派。

### 适用场景
- 分配工作任务给团队成员
- 将需要人工处理的环节交给指定人员
- 传达执行指令或安排日程

### 关键词
`分派` `委派` `分配` `交给` `让` `安排` `派给` `下发` `布置` `叫`

### 匹配联系人
`contacts.json` 中的 `subordinates` 列表

### 特殊消歧规则
- "交给" + Agent 名称（如"代码审查Agent"）→ 归为 RIGHT，而非 DOWN
- "让XX Agent处理" → 归为 RIGHT
- "完成" 出现在含目标人物的句子中 → 主方向为 DOWN，"完成"成为内容的一部分

---

## ← LEFT — 执行/存档（Execute/Archive）

### 语义定义
任务处理满意后，将成果存档或标记为完成。属于任务状态管理，不涉及人际分发。

### 适用场景
- Agent 完成工作后，用户审阅满意，标记为完成
- 阶段性任务结束，归档当前工作
- 关闭或终结一个任务项

### 关键词
`存档` `标记完成` `归档` `完成` `关闭任务` `标记` `结束`

### 匹配联系人
无。LEFT 方向不涉及人际分发，仅做任务状态变更。

### 特殊消歧规则
- "完成" 独立使用或与任务内容结合（如"完成这个任务"）→ LEFT
- "完成" 出现在含目标人物的句子中（如"让小王完成报告"）→ 主方向为 DOWN
- "完成了，存档" → LEFT

---

## → RIGHT — Agent 分派（Delegate to Agent）

### 语义定义
将任务分配给 AI Agent 处理，让机器继续工作。Agent 完成后将结果反馈给用户，用户再决定下一步流向。

### 适用场景
- 将文档交给润色 Agent 优化
- 将代码交给审查 Agent 检查
- 将数据交给分析 Agent 处理
- 任何需要 AI 继续加工的任务

### 关键词
`交给...Agent` `Agent处理` `AI处理` `智能处理` `机器处理` `自动处理` `让...分析`

### 匹配联系人
`contacts.json` 中的 `agents` 列表

### 特殊消歧规则
- "交给" 后面的目标包含 "Agent" 或 "AI" 关键词 → 优先归为 RIGHT
- 目标在 agents 列表中找到 → RIGHT
- 目标在 subordinates 列表中找到 → DOWN
- 两者都找不到时，检查文本是否含 "Agent"/"AI" 字样 → 是则 RIGHT，否则 DOWN

---

## 中 CENTER — 中枢（Comm Hub）

### 语义定义
当用户输入不包含任何方向关键词时，系统默认为 CENTER 状态。此时用户处于决策中枢，输入被视为信息查询或待定意图。

### 适用场景
- 用户询问状态（"现在进度怎么样了"）
- 用户浏览信息（"显示今天的任务列表"）
- 语音输入不完整或模糊

### 关键词
无特定关键词。所有未被其他方向匹配的输入归为 CENTER。

### 匹配联系人
无。

---

## 多方向歧义处理

当一句话中包含多个方向关键词时：

1. **优先采用最早出现的方向关键词**作为主方向
2. 被覆盖的方向关键词记录在 `parse_metadata.ambiguities` 中
3. 示例："汇报给张总，然后分派给小王执行" → 方向 UP，"分派"记录为歧义

### 常见歧义组合

| 输入特征 | 判定方向 | 理由 |
|----------|---------|------|
| "交给" + Agent 名称 | RIGHT | 目标类型优先于关键词 |
| "交给" + 人名 | DOWN | 目标类型优先于关键词 |
| "完成" + 目标人物 | DOWN | 委派语义覆盖状态管理 |
| "完成" 独立使用 | LEFT | 纯状态管理 |
| 多方向关键词 | 最先出现者 | 时间优先级 |
